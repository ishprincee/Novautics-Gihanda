Novautics-Gihanda Concept
1. Project Overview
Novautics-Gihanda is an innovative business specializing in video intercom installation and remote gate access solutions. We provide customers with a secure and user-friendly system that allows them to open and close gates remotely, monitor visitors via video intercoms, and manage access efficiently.
The solution targets residential buildings, commercial properties, and gated communities, improving security, convenience, and access control through modern technology.

2. Business Objectives
•	Enhance property security using video intercom systems
•	Allow users to remotely open and close gates from their devices
•	Reduce the need for physical presence at access points
•	Improve visitor monitoring and communication
•	Provide reliable and scalable access control solutions

3. Stakeholders
•	Business Owner (Novautics-Gihanda) – Oversees service delivery and growth
•	System Engineer – Manages system set-up, configurations and access rights
•	Property Owners / Managers – Control access to their premises
•	End Users (Residents / Staff) – Use the system to communicate and control gates
•	Installers / Technicians – Install and maintain intercom and gate systems




4. Use Cases
UC-01: Video Intercom Communication
Actor: User
Description: The user receives a video call from a visitor at the gate.
Outcome: The user can see and communicate with the visitor in real time.

UC-02: Remote Gate Control
Actor: Authorized User
Description: The user opens or closes the gate remotely using a mobile or desktop interface.
Outcome: Secure and convenient gate access.

UC-03: Visitor Access Authorization
Actor: User
Description: The user verifies a visitor and grants or denies entry.
Outcome: Controlled and safe visitor access.

UC-04: User Management
Actor: Administrator
Description: The administrator adds, updates, or removes authorized users.
Outcome: Proper access control and system security.

UC-05: System Monitoring and Maintenance
Actor: Administrator / Technician
Description: Monitor system health and perform maintenance.
Outcome: Reliable system operation.

5. User Stories
User Story 1 – Receive Video Call
As a property owner, we want to see and talk to visitors through a video intercom so that we can verify who is requesting access.

User Story 2 – Open Gate Remotely
As an authorized user, we want to open the gate remotely so that visitors can enter without us being physically present.

User Story 3 – Manage Access Rights
As an System Engineer, we want to manage user permissions so that only authorized users can have the right control and access to the gate.

User Story 4 – Improve Security
As a property manager, we want controlled gate access so that the premises remain secure.

User Story 5 – System Reliability
As a technician, we want to monitor and maintain the system so that downtime is minimized.

6. Assumptions and Constraints
•	Users have access to a smartphone or computer
•	Stable internet connection is required for remote access
•	Security and privacy standards must be respected
•	Hardware installation must comply with local regulations
